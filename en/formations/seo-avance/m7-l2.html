Strategie de migration SEO sans perdre du trafic

Module 7 · Lesson 2/8 ⏱ 23 min Progress : 81%

Une migration de site (refonte, changement de domaine, plateforme, structure URL) est l'un des risques SEO les plus eleves. Une migration mal preparee peut effacer 50-90 % du trafic en 7 jours. Cette lecon vous donne le protocole pour migrer sans perte.

Sur les 14 migrations conduites par notre equipe depuis 2022, le bilan : 11 ont conserve > 95 % du trafic, 3 ont gagne du trafic (+15 a +40 %), 0 a subi une perte significative. Voici la methode.

Types de migrations

1. Migration de plateforme

Ex : WordPress vers Webflow, Shopify vers Magento. Risque moyen si URLs preservees.

2. Migration de domaine

Ex : olddomain.com vers newdomain.com. Risque eleve, 6-9 mois de transition.

3. Refonte structure URLs

Ex : /blog/article-titre vs /article-titre. Risque eleve si non geree.

4. Refonte design / contenu

Conservation URLs mais changement majeur. Risque moyen si E-E-A-T preserve.

5. Passage HTTP vers HTTPS

Migration "obligatoire". Risque faible si bien fait.

6. Migration multilingue

Ex : ajout versions FR/EN. Risque moyen, hreflang critique.

Le protocole en 8 phases

Phase 1 : audit pre-migration (J-60)

  • Crawl complet de l'ancien site (Screaming Frog).
  • Export liste URLs avec trafic (GA4 + GSC, 12 mois).
  • Export backlinks (Ahrefs, Semrush).
  • Export positions actuelles.
  • Capture baseline KPIs : trafic, conversions, RDs, DR.

Phase 2 : plan de redirection (J-45)

Mapping 1:1 entre anciennes et nouvelles URLs :

  • Tableau Excel avec ancienne URL > nouvelle URL > type de redirection (301).
  • Toutes les pages avec trafic doivent etre redirigees.
  • Pages obsoletes : 410 Gone.
  • Eviter chaines de redirection (A > B > C).

Phase 3 : preparation environnement staging (J-30)

  • Nouveau site monte sur staging.staging-domain.com.
  • Protection par mot de passe (robots.txt disallow + auth HTTP).
  • Verifier toutes les pages critiques fonctionnent.

Phase 4 : audit technique staging (J-21)

  • Crawl Screaming Frog complet.
  • Verification title, meta, H1, canonical, robots.
  • Schema.org valide.
  • Hreflang correct (si multilingue).
  • Sitemap.xml propre.
  • Core Web Vitals OK.

Phase 5 : tests de redirection (J-14)

  • Configurer les 301 sur le staging.
  • Verifier echantillon 50 redirections : 301 directe vers bonne destination.
  • Outils : Httpstatus.io, Redirect Path (Chrome).

Phase 6 : Go-Live (J-Day)

  • Choisir un mardi/mercredi (eviter weekend).
  • Eviter periode fortes promotions.
  • Backup complet ancien site.
  • Mise en ligne avec redirections actives immediatement.
  • Soumettre nouveau sitemap a GSC.
  • "Address Change" tool si changement de domaine.

Phase 7 : monitoring post-migration (J+1 a J+30)

  • J+1 : verification 100 % des redirections critiques.
  • J+3 : crawl du nouveau site complet.
  • J+7 : check GSC Index Coverage (chute normale + recuperation).
  • J+14 : verification trafic GA4 vs baseline.
  • J+30 : rapport complet post-migration.

Phase 8 : optimisation post-migration (J+30 a J+90)

  • Recuperer positions perdues sur top mots-cles.
  • Pousser sitemap et URL inspection sur pages cles.
  • Refresh content si necessaire.
  • Update backlinks externes si possible.

Les redirections : LE point critique

Bonnes pratiques

  • 301 (permanent) et non 302 ou 307.
  • 1:1 mapping : chaque URL importante vers son equivalent precis.
  • Pas de redirect vers homepage en masse (signal weak).
  • Pas de chaines : eviter A > B > C, faire A > C directement.
  • Conserver les ancres #section si pertinent.

Outils

  • Redirection plugin (WordPress).
  • .htaccess (Apache).
  • nginx.conf (Nginx).
  • Vercel.json / Netlify _redirects (JAMstack).

Attente realiste post-migration

  • Semaine 1 : chute trafic 10-30 % normale (Google reindexe).
  • Semaine 2-4 : recuperation progressive.
  • Mois 2-3 : retour 90-100 % du trafic.
  • Mois 4-6 : potentiellement plus si optimisations.
"Toute migration coute 4-12 semaines de SEO. Anticipez ce delai dans votre planning business." — Aleyda Solis

Erreurs catastrophiques a eviter

  1. Pas de redirections : -100 % du trafic en 30 jours.
  2. Robots.txt disallow oublie depuis staging : tout deindexe.
  3. Noindex meta oublie : meme effet.
  4. HTTPS sans redirection HTTP : duplicate content.
  5. Sitemap non mis a jour : Google ne trouve pas les nouvelles URLs.
  6. Change Address dans GSC oublie : transition tres lente.

Cas concret : migration WordPress vers Webflow

Pour une PME beninoise (340 pages) :

  • Audit pre : 12 400 sessions organiques/mois.
  • Migration en 6 semaines avec protocole strict.
  • J+30 post-migration : 11 200 sessions (-10 %).
  • J+90 : 14 800 sessions (+19 % grace a perf amelioree).

FAQ

Quand faut-il faire une migration ?

Quand le ROI estime > 50 % de perte trafic potentielle. Refonte uniquement pour SEO peu rentable, sauf si site tres degrade.

Combien coute une migration SEO ?

3 000-25 000 EUR pour le suivi SEO selon taille. Hors developpement de la nouvelle plateforme.

Faut-il informer Google de la migration ?

Oui via Change Address Tool si nouveau domaine. Rapide a faire et essentiel.
